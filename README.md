# Mortgage Affordability Automation

Browser automation service for UK mortgage intermediary affordability calculators.

The project accepts lender-ready JSON, selects the correct lender adapter, fills the lender affordability calculator with Playwright, extracts the result, and returns structured JSON with optional screenshot evidence.

## Supported Lenders

The current adapter registry includes:

- Barclays
- Halifax
- HSBC
- Kensington
- Nationwide
- NatWest
- Santander
- Skipton
- Virgin Money

Each lender adapter lives under `src/adapters/<lender>/` and compiles to `dist/adapters/<lender>/`.

## Project Structure

```text
src/        TypeScript source code. Make changes here.
dist/       Compiled JavaScript output generated by npm run build.
samples/    Lender-ready JSON examples for each supported lender.
docs/       Field maps, implementation notes, and handoff documentation.
scripts/    Helper scripts for samples and attached browser mode.
```

`src/` is the source of truth. Do not manually edit `dist/` for permanent changes because it is regenerated from `src/`.

## How Build Output Works

This is a TypeScript project. The calculator automation code is written in `src/` as `.ts` files.

Running:

```powershell
npm run build
```

executes:

```powershell
tsc -p tsconfig.json
```

and generates matching JavaScript files in `dist/`.

Examples:

```text
src/adapters/santander/adapter.ts    -> dist/adapters/santander/adapter.js
src/adapters/halifax/adapter.ts      -> dist/adapters/halifax/adapter.js
src/adapters/virgin-money/adapter.ts -> dist/adapters/virgin-money/adapter.js
```

So a fresh clone can recreate `dist/` with:

```powershell
npm install
npm run build
```

## Input Model

Inputs are expected to be lender-ready JSON documents. Raw fact-find conversion is intentionally outside this project.

The service owns:

- lender adapter selection
- input validation
- browser automation
- calculator field interaction
- result extraction
- evidence capture
- structured success/failure output

## Sample Usage

Run one sample directly through the TypeScript CLI:

```powershell
npx tsx src/cli.ts samples/santander/01-purchase-employed-standard.json
```

Run compiled JavaScript after building:

```powershell
npm run build
node dist/cli.js samples/santander/01-purchase-employed-standard.json
```

The output shape is:

```json
{
  "lender": "santander",
  "status": "success",
  "maximumBorrowing": 250000,
  "monthlyPayment": null,
  "messages": [],
  "evidence": {
    "screenshotPath": "./artifacts/screenshots/example.png",
    "timestamp": "2026-05-22T00:00:00.000Z"
  }
}
```

Values vary by lender calculator response and input data.

## Browser Execution Modes

The default mode is `managed`, where Playwright launches a browser owned by the service.

Some lender sites may reject fresh automation contexts but work in an existing trusted browser session. For those cases, use attached browser mode.

Start a visible browser with remote debugging:

```powershell
npm run attached:browser
```

Open:

```text
http://127.0.0.1:9222/json/version
```

Copy `webSocketDebuggerUrl`, then run:

```powershell
$env:BROWSER_EXECUTION_MODE="attached"
$env:BROWSER_WS_ENDPOINT="ws://127.0.0.1:9222/devtools/browser/..."
$env:HEADLESS="false"
npm run build
node dist/cli.js samples/halifax/test-case-1.json
```

Credentials and broker portal sessions should stay in the browser profile. Do not commit credentials or secrets to the repository.

## Configuration

Environment variables:

```text
BROWSER_EXECUTION_MODE   managed or attached
BROWSER_WS_ENDPOINT      Chrome DevTools WebSocket URL for attached mode
HEADLESS                 false to show the browser
AUTOMATION_TIMEOUT_MS    default 60000
SCREENSHOT_DIR           default ./artifacts/screenshots
PORT                     API port, default 3000
```
